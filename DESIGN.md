# JOJO Design System

版本：1.3  
定位：叫叫产品设计语言与工程标准化规范  
适用范围：叫叫品牌官网、叫叫 APP、绿豆 APP、学科落地页、学习流程 demo、设计系统验证 demo

## 1. 项目目标

这个项目用于打磨 JOJO 产品设计语言的工程标准化能力：通过制作不同场景的 demo，验证品牌视觉、组件状态、动效反馈、响应式布局和可访问性是否可以被稳定复用。

设计文档不只记录“长什么样”，还要记录：

1. 可复用的设计 token。
2. 组件状态与反馈模型。
3. 动效参数与降级策略。
4. 页面和 demo 的验收标准。
5. 设计与开发之间可以直接对齐的实现边界。

## 2. 品牌定位

叫叫是面向 4-12 岁儿童的儿童成长数字内容平台，覆盖阅读、英语、思维、美育、小作家等多个学科产品。品牌以孩子兴趣为基础，用有趣有效的数字内容和陪伴式服务，让孩子更爱学。

品牌关键词：

| 关键词 | 设计含义 |
| --- | --- |
| 温暖 | 明亮、亲近、鼓励式表达，避免压迫感 |
| 童趣 | 有角色、有故事、有反馈，但不低幼化 |
| 清晰 | 信息层级明确，阅读负担低，核心行动明确 |
| 专业 | 内容可信、承诺克制、效果表达具体 |
| 包容 | 同时照顾孩子的探索感与家长的信任感 |

## 3. 产品矩阵

叫叫教育旗下主要产品共享同一套底层设计语言，在主色、按钮形态和内容气质上保持差异。

| 产品 | 定位 | 主色 | 按钮圆角 | 核心气质 |
| --- | --- | --- | --- | --- |
| 叫叫 APP | 多学科儿童成长数字内容平台 | `#FCDA00` | `999px` | 活泼、温暖、全面 |
| 绿豆 APP | 儿童英语学习产品 | `#78C562` | `10px` | 专注、稳重、成长 |
| 品牌官网 | 对外品牌与内容能力展示 | 黄色为主，学科色辅助 | `999px` / `16px` | 可信、有趣、内容平台感 |

统一设计语言可以降低用户学习成本，强化 JOJO 家族识别；主色和形态差异用于表达产品个性。

## 4. 设计原则

### 4.1 有趣但不低幼

学习体验应该能激发孩子的内在驱动力。视觉可以柔和、粉嫩、明亮，角色和场景可以有叙事，但不要过度游戏化或营销化。

设计判断：

| 视角 | 应该做到 |
| --- | --- |
| 孩子 | 一眼被吸引，有角色、有故事、有即时反馈 |
| 家长 | 孩子会喜欢，同时品相精良、不廉价、不夸张 |

### 4.2 专业但不冰冷

教育产品需要建立家长信任。专业感来自清晰的信息架构、真实的效果表达和高质量内容，而不是冷色调或复杂术语。

设计判断：

1. 不使用夸大承诺。
2. 用数据、案例、报告、来源说明表达可信度。
3. 内容模块要有平台感，不像单一活动页。
4. AI、测评、报告等前沿功能表达要克制、具体。

### 4.3 先反馈，再执行动作

参考 `Design-mikey.MD` 的动效体系，JOJO 的交互也采用“先确认意图，再执行动作”的思路。重要跳转、下载、提交、打开外链等动作可以使用轻量二次确认或明确的 pressed 状态，降低误触。

### 4.4 反馈分层

一次好用的反馈至少由三层中的两层组成：

| 层级 | 职责 | 示例 |
| --- | --- | --- |
| 背景 | 表达状态 | 彩虹柔色、浅色填充、选中底色 |
| 内容 | 表达意图 | 文字变化、动作词、箭头 |
| 物理感 | 表达触感 | 缩放、上浮、弹簧回弹 |

### 5.17 色阶说明

每组颜色的 1-6 色阶遵循以下规律：

| 色阶 | 用途 | 亮度特征 |
| --- | --- | --- |
| 1 | 极浅背景 | 最高亮度 |
| 2 | 浅色背景、次级元素 | 高亮度 |
| 3 | hover、轻强调 | 中高亮度 |
| 4 | 主色调、品牌强调 | 标准亮度 |
| 5 | 按压状态、深色元素 | 较低亮度 |
| 6 | 深色文字、深色背景 | 最低亮度 |

使用建议：
- 背景使用 1-2 色阶
- 交互态使用 3-4 色阶
- 文字和深色状态使用 5-6 色阶
- 保持同色系内的对比度，确保可读性

## 9. 组件规范

### 9.1 按钮

| 类型 | 背景 | 文字 | 圆角 | 反馈 |
| --- | --- | --- | --- | --- |
| 叫叫主按钮 | `yellow-4` | `yellow-6` | `999px` | hover 变亮，active 轻缩放 |
| 叫叫次按钮 | `yellow-2` 或白底描边 | `yellow-6` / `gray-6` | `999px` | hover 描边加深 |
| 绿豆主按钮 | `green-4` | `white` | `10px` | hover 变亮，保留 2px 深绿边 |
| 官网文字按钮 | 透明 | `gray-6` | `999px` | hover 柔色底，文字上移 |

最小触摸热区：`44px`。按钮内文案要短，动词明确。

### 9.2 二次确认按钮

适用：下载、打开外链、提交、发送、跳转离开页面。

状态机：

| 状态 | 触发 | 视觉 | 行为 |
| --- | --- | --- | --- |
| Default | 初始 | 普通按钮 | 等待点击 |
| Pressed | 点击瞬间 | 内容层 `press-jump` 或 `press-shrink` | 触感反馈 |
| Selected | 首次点击 | 柔色流动背景、文字变深、出现动作词 | 不执行最终动作 |
| Confirm | 2 秒内再次点击 | 再次按压反馈 | 执行最终动作 |
| Fading | 失焦、超时或外部点击 | 背景与动作词 1000ms 淡出 | 清理状态 |
| Disabled | 不可用 | 透明度降低，禁用焦点 | 不响应 |

默认确认窗口：`2000ms`。高风险动作可以延长到 `3000ms`。

### 9.3 信息卡片

卡片只用于独立完整的信息单元。页面大区块不要全部做成浮动卡片。

内容卡片建议：

1. 背景白色。
2. 圆角 `16px` 或 `24px`。
3. 轻描边优先于重阴影。
4. hover 只做轻微上浮，位移不超过 `4px`。

## 10. 动效系统

动效继承 `Design-mikey.MD` 的核心思路：短、轻、有物理感；先表达状态，再执行动作。JOJO 场景的动效要更温暖、更柔和，避免强烈晃动或过度弹跳。

### 10.1 Motion token

```css
:root {
  --motion-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
  --motion-standard: cubic-bezier(0.22, 1, 0.36, 1);
  --motion-modal: cubic-bezier(0.16, 1, 0.3, 1);
  --motion-quick: 160ms;
  --motion-press: 280ms;
  --motion-enter: 600ms;
  --motion-fade: 1000ms;
  --motion-confirm-window: 2000ms;
}
```

### 10.2 核心动效

| 名称 | 时长 | 缓动 | 使用场景 |
| --- | --- | --- | --- |
| `spring-scale-in` | `600ms` | `--motion-spring` | hero 元素、卡片、学科模块入场 |
| `soft-slide-up` | `520ms` | `--motion-standard` | 官网段落进入视口 |
| `press-jump` | `280ms` | `--motion-spring` | 趣味按钮、学科卡点击 |
| `press-shrink` | `160ms` | `ease-out` | 普通命令按钮 |
| `focus-color` | `2000ms` | `ease-in-out infinite` | 二次确认选中态 |
| `float-soft` | `2400ms` | `ease-in-out infinite` | 品牌角色、轻量装饰 |
| `shake-feedback` | `500ms` | `cubic-bezier(.36,.07,.19,.97)` | 错误、不可用反馈 |

### 10.3 彩虹柔色聚焦

彩虹柔色不是装饰，而是选中态反馈。适用场景：

1. 重要按钮首次点击。
2. 可选择卡片。
3. 官网 tab 或轮播当前项。
4. Demo 中需要强调“已选中”的互动模块。

推荐关键帧：

```css
@keyframes jojo-focus-color {
  0%, 100% { background: #FCF7BD; }
  25% { background: #D9ECFF; }
  50% { background: #FFE8CC; }
  75% { background: #E7DDFF; }
}
```

文字颜色统一使用 `#3A3A3A` 或 `#404040`，不要在浅色流动背景上使用白字。

### 10.4 入场节奏

官网和 demo 的入场推荐：

| 元素 | 延迟 |
| --- | --- |
| 主标题 | `0ms` |
| 副文案 | `80ms` |
| 主按钮 | `140ms` |
| 视觉资产 | `180ms` |
| 列表卡片 | `index * 60ms` |

列表项延迟不超过 `360ms`，避免页面变慢。

### 10.5 文字切换

短标签、按钮动作词、tab 标题可以使用从下向上的文字切换：

| 阶段 | 时长 | stagger | 说明 |
| --- | --- | --- | --- |
| Exit | `220ms` | `24ms/字` | 旧文本上移淡出 |
| Gap | `30ms` | 无 | 切换内容 |
| Enter | `360ms` | `48ms/字` | 新文本从下进入 |

仅用于 12 个中文字符以内的短文本。长文案使用整句 fade。

### 10.6 Reduced motion

必须提供动效降级：

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 1ms !important;
    animation-iteration-count: 1 !important;
    scroll-behavior: auto !important;
    transition-duration: 1ms !important;
  }
}
```

降级后保留颜色、透明度和状态文案，移除循环、弹跳和大位移。

## 11. 可访问性

1. 所有核心交互最小高度 `44px`。
2. 键盘可达，`Enter` / `Space` 可触发按钮。
3. `focus-visible` 必须有可见描边。
4. 禁用态使用 `disabled` 或 `aria-disabled`，不能只靠颜色。
5. 无限动画必须在 `prefers-reduced-motion` 下关闭或弱化。
6. 重要信息不能只依赖颜色区分，要有文字或图标辅助。
7. 来源、脚注、证书说明字号不低于 `12px`，行高不低于 `150%`。

## 12. 官网设计规范

品牌官网应服务两个目标：让家长和外部合作方快速理解“叫叫是谁”，同时展示内容、教研、科技和行业影响力。

### 12.1 首屏

首屏必须明确出现：

1. 叫叫 logo 或品牌名。
2. 品牌核心描述：儿童成长数字内容平台。
3. 核心主张：让学习变得有趣有效 / 让孩子更爱学。
4. 至少一个真实品牌资产：logo、APP icon、角色形象、产品截图或官网内容图。

H1 建议使用品牌名或直接品类，不把价值主张堆成标题。

### 12.2 内容结构

推荐顺序：

1. 首屏品牌叙事。
2. 关键数据：18 年、连续五年、千万家庭、城市与国家覆盖。
3. 关于叫叫：有趣有效的数字内容。
4. 四大优势：用户覆盖、教研测评、家庭场景、内容经验。
5. 产品矩阵：阅读、美育、思维、小作家、英语等。
6. 团队与研究：教育、出版、动画、应试、游戏、科技人才。
7. 行业创新与报告。
8. 页脚联系方式与 demo 链接。

### 12.3 语气

官网文案要可信、清晰、克制：

| 推荐 | 避免 |
| --- | --- |
| 有趣有效的数字内容 | 颠覆式改变学习 |
| 陪伴千万孩子成长 | 保证成绩提升 |
| 以孩子兴趣为基础 | 秒变学霸 |
| 研究报告、来源说明 | 模糊权威背书 |

Demo 页面优先在 `:root` 中声明 token，再使用变量。避免颜色散落在页面中。

### 13.3 Demo 验收清单

每个 demo 至少检查：

1. 是否遵守品牌色和学科色规则。
2. 是否使用正确字体场景。
3. 移动端和桌面端是否都无溢出。
4. 核心按钮是否有 hover、active、focus-visible、disabled 或可解释的状态。
5. 动效是否短、轻、有反馈，并支持 reduced motion。
6. 重要文案是否真实、不夸大。
7. 图片、logo、字体路径是否可加载。
8. 文字不遮挡、不挤压、不因 hover 改变布局。

## 14. Do / Don't

### Do

1. 通用场景使用 APP 主色，学科场景使用学科色。
2. 使用柔和、明亮、粉嫩但高级的颜色。
3. 使用留白、轻描边和彩色区块建立层级。
4. 序号和品牌数据优先使用 Mohr Rounded。
5. 交互先给反馈，再执行关键动作。
6. 为 demo 写清楚状态、动效和验收标准。
7. 文案表达具体效果和真实来源。

### Don't

1. 不使用灰暗、刺眼或单一浓度的颜色铺满页面。
2. 不滥用灰底白卡和卡片套卡片。
3. 不随意使用学科色做通用按钮。
4. 不在日常 UI 场景使用 JOJO Kaiti。
5. 不做没有状态意义的循环动效。
6. 不使用夸大承诺或强营销话术。
7. 不让视觉趣味牺牲可读性和专业感。
